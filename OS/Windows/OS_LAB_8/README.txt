OS_LAB_8 - отображаемые в память файлы

Запуск в таком порядке
1.PRODUCER
2.CODER
3.DECODER
4.CONSUMER

сообщение в input.txt PRODUCER'a не делайте большим (лучше вообще оставить так)
Когда по очереди будете запускать проги, дайте каждой запуститься, буквально 1-2 сек
Но не слишком долго, в конце каждой написано что они работают по 10-15 сек, поэтому
если не успеваете (ПК лагает и тд) то можете поставить задержку больше

PRODUCER - создает сообщение из input.txt файла и отсылает по pipe CODER
CODER    - кодирует сообщение из pipe (в обратном порядке) и отображает в память по файлу DECODER
DECODER  - декодирует сообщение из отображ.файла и отсылает по pipe CONSUMER
CONSUMER - получает сообщение из pipe и записывает его в файл
